# AI Content Telegram Bot

AI Content Telegram Bot — это проект, предназначенный для автоматической генерации контента длиной 250 слов с использованием эмодзи, что идеально подходит для Telegram-каналов или блогов.

## Основные функции
- **Генерация постов**: Создание качественного контента с использованием AI (Perplexity Sonar).
  - 📝 **Текстовые посты**: Генерация постов только с текстом (200-300 слов).
  - 🖼️ **Посты с изображениями**: Генерация постов с текстом и до 3 релевантных изображений (через Pexels API).
- **Статистика (только для администраторов)**: Отслеживание использования бота.
  - Общее количество сгенерированных постов (с разделением по типам).
  - Количество активных пользователей.
  - Популярные темы постов.
- **RAG (Retrieval-Augmented Generation)**: Использование пользовательских документов для улучшения качества контента.
- **Мультиязычность**: Автоматический перевод контента (RU/EN).
- **Автопостинг**: Автоматическая публикация постов в канал каждые 6 часов.

## Архитектура (Модульная структура)
Проект использует модульную архитектуру для улучшения поддерживаемости и тестируемости:
- **config.py** - Управление конфигурацией и переменными окружения
- **logger_config.py** - Настройка логирования с фильтрацией чувствительных данных
- **api_client.py** - Клиент Perplexity API с повторными попытками
- **translation_service.py** - Сервис перевода с определением языка
- **rag_service.py** - Поиск по векторной базе данных
- **bot_statistics.py** - Отслеживание статистики использования
- **image_fetcher.py** - Получение изображений через Pexels API
- **bot.py** - **Основной (канонический) модуль с обработчиками Telegram — именно его следует запускать**
- **main.py** - Обёртка для обратной совместимости, делегирует запуск в `bot.py`

## Инструкция для запуска
1. Убедитесь, что ваш файл `.env` содержит следующие переменные:
   ```bash
   BOT_TOKEN=ваш_токен
   PPLX_API_KEY=ваш_api_key
   CHANNEL_ID=@ваш_канал
   PEXELS_API_KEY=ваш_pexels_api_key  # Опционально, для постов с изображениями
   ADMIN_USER_IDS=123456789,987654321  # Telegram ID администраторов через запятую
   RAG_ENABLED=true  # Опционально, для включения/отключения RAG (default: true)
   ```
   См. `.env.example` для примера.

2. **Настройка Pexels API (для постов с изображениями):**
   - Зарегистрируйтесь на [Pexels](https://www.pexels.com/api/)
   - Получите бесплатный API ключ
   - Добавьте ключ в `.env` файл как `PEXELS_API_KEY`
   - Бесплатный план: 200 запросов в час

3. Установите зависимости:
   ```bash
   # Базовые зависимости (быстрая установка)
   pip install -r requirements.txt
   
   # Опционально: RAG зависимости (для улучшения качества контента)
   # Устанавливайте только если планируете использовать RAG функции
   pip install -r requirements-rag.txt
   ```
   
   **Настройка RAG (Retrieval-Augmented Generation):**
   
   RAG улучшает качество контента, используя вашу базу знаний. По умолчанию включен (`RAG_ENABLED=true`).
   
   **Шаги для включения RAG:**
   1. **Установите зависимости:** 
      ```bash
      pip install -r requirements-rag.txt
      ```
      (⚠️ ~2GB, требует время на установку)
   
   2. **Добавьте файлы знаний:**
      - Разместите ваши документы (`.txt`, `.md`, `.pdf`) в папку `knowledge/`
      - Примеры: инструкции компании, спецификации продуктов, руководства по стилю
   
   3. **Запустите бота:**
      ```bash
      python bot.py
      ```
      Проверьте лог: `✅ RAG service initialized successfully`
   
   4. **Если появляется ошибка** "No module named 'langchain_community'":
      - Установите RAG-зависимости: `pip install -r requirements-rag.txt`
   
   **Отключение RAG (для более быстрого запуска):**
   - В `.env` файле установите: `RAG_ENABLED=false`
   - Бот будет работать без RAG, с меньшим использованием памяти
   
   **Переопределение модели эмбеддингов (опционально):**
   - По умолчанию: `sentence-transformers/all-MiniLM-L6-v2`
   - Для изменения добавьте в `.env`: `EMBEDDINGS_MODEL=your-model-name`

4. Запустите локально приложение:
   ```bash
   python bot.py
   ```
   > **Единая точка входа:** `bot.py` — канонический стартовый файл для локальной разработки, Railway и любого другого деплоя.
   > `main.py` сохранён для обратной совместимости и просто делегирует запуск в `bot.py`.
   > Railway-команда прописана в `Procfile`: `worker: python bot.py`.
